# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Repo Is

Wiring harness and schematic data for Cummins diesel engine swaps into Ford trucks ("Fummins") and related projects. All diagrams are generated by [WireViz](https://github.com/wireviz/WireViz) from YAML source files.

## Building

Each sub-project has its own `build.sh` that runs WireViz on the local YAML file:

```sh
cd harnesses/00-02-cummins-engine && ./build.sh
# equivalent to: wireviz harness.yml
```

Generated outputs (`.gv`, `.svg`, `.png`, `.html`, `.bom.tsv`) are committed alongside the YAML source — always regenerate and commit them after changing a YAML file.

Install WireViz if needed:
```sh
pip install wireviz
```

## Directory Structure

| Directory | Purpose |
|-----------|---------|
| `harnesses/` | Full engine wiring harnesses (e.g., 00-02 Cummins ECU harness) |
| `hookups/` | Point-to-point hookup diagrams for specific components (VP44, H-pump) |
| `Fummy/` | Adapter cables for the Fummy build (Ford pedal → ODBW, etc.) |
| `odbw/` | Open-source Drive-By-Wire controller schematic and pedal harness |

## WireViz YAML Format

Each YAML file has three top-level sections:

```yaml
connectors:
  myConnector:
    type: Male Plug
    mpn: DT06-6S          # manufacturer part number
    manufacturer: Deutsch
    pinlabels: [A, B, C]
    image:
      src: connectors/image.webp
    notes: any notes here

cables:
  myCable:
    colors: [RD, BK, WH]  # wire color codes per conductor
    gauge: 20 AWG

connections:
  -
    - connectorA: [1, 2]   # pin numbers (1-indexed)
    - myCable: [1, 2]      # conductor numbers (1-indexed)
    - connectorB: [3, 4]
```

**Wire color codes**: `RD`=red, `BK`=black, `WH`=white, `GN`=green, `BU`=blue, `YE`=yellow, `GY`=grey, `VT`=violet, `OG`=orange, `BN`=brown. Two-letter combos indicate striped wires (e.g., `RDWH` = red/white).

**Part numbers**: Use `mpn` (manufacturer part number) on connectors when known. Leave as `???` when unknown.

## Adding a New Harness or Cable

1. Create a new directory under the appropriate top-level folder.
2. Write the YAML source file (name it `harness.yml`, `schematic.yml`, or descriptively).
3. Add a `build.sh`:
   ```sh
   wireviz <filename>.yml
   ```
4. Run `build.sh` and commit all generated files with the YAML.
